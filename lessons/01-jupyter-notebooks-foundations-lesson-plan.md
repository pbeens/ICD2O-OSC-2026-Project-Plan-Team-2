## Lesson Role In The Pathway

This is the front-facing anchor document for Lesson 01. It gives the big picture for the transition from conceptual understanding into notebook workflow, so students see Jupyter as part of a larger analysis process rather than as isolated software steps.

- B2.1 use file management techniques to organize, edit, and share files.
- B2.2 identify and use effective practices when learning to use new software.
- C3.1 analyze existing code to understand components and outcomes.
- C3.4 use external modules or libraries in a program.

## Learning Goals

Students will be able to:

- Describe what a Jupyter Notebook is and why it is useful for analysis.
- Distinguish markdown cells from code cells.
- Run cells in order and explain `Shift+Enter` workflow.
- Import libraries in a notebook and explain why imports are usually near the top.
- Use simple output cells to confirm code is working.

## Teacher Prep And Opening Sequence

1. Review this lesson plan first for the workflow goals, troubleshooting focus, and the role of Lesson 01 as the bridge between Lesson 00 concepts and later full notebook analysis.
2. Read through the companion tutorial: [01-jupyter-notebooks-foundations-tutorial.md](01-jupyter-notebooks-foundations-tutorial.md) before class so the definitions of notebook, cell, library, import, DataFrame, method, and property are ready for modelling.
3. Open Jupyter Notebook or Google Colab before class and prepare a blank starter notebook students can copy from or follow live.
4. Confirm students know how they will access the notebook environment during class.
5. Make explicit to students that this lesson prepares them for the more complete notebook and assignment work in the next lesson sequence.
6. Begin by showing the difference between a plain script and a notebook before students create their own cells.

## Activity List

1. Opening comparison: students compare a plain script view to a notebook view and discuss which is easier for learning.
2. Notebook setup: students open Jupyter or Colab and create a new notebook.
3. Cell practice: students add one markdown cell and one code cell, then run them in order.
4. Vocabulary mini-lesson: teacher explains import, library, DataFrame, method, property, kernel/runtime, and execution order using the companion tutorial.
5. Guided mini-sequence: students import libraries, create a tiny DataFrame, preview rows, inspect columns, and make a simple chart.
6. Troubleshooting practice: students restart the kernel/runtime and rerun all cells from top to bottom.
7. Reflection/checkpoint: students explain one execution-order mistake and how to fix it.

## Lesson Body

This lesson teaches notebook mechanics and workflow so students can focus on analysis in Lesson 02.

### Engage

Show two screenshots: plain script vs notebook with text + code + output. Ask which format is easier for learning and why.

### Explore

Students create a notebook, add one markdown cell and one code cell, then run cells in order. They practise restarting kernel/runtime and re-running top-to-bottom.

### Explain

Explain notebook structure (cells), execution order, why imports are grouped at the top, and how outputs reflect the most recently run state.

### Elaborate

Students run a short mini-sequence: import libraries, create small DataFrame, inspect `head()`, print column names, and make a tiny chart.

### Evaluate

Evidence from observations, conversations, and student products. Checkpoint: student can correctly create markdown/code cells, run all cells in order, and explain one execution-order mistake and fix.

## Homework

In 5-7 sentences, explain how Jupyter notebooks help with reproducibility and communication in data analysis.

## Comments

Keep this lesson procedural and confidence-building. Use short cells and immediate feedback to reduce beginner frustration. The lesson plan is the front-facing lesson document, the tutorial should be read before class, and the live notebook environment should only be opened after that preparation is complete. This lesson should be framed as preparation for the later notebook-based assignment work rather than as an isolated software lesson.
