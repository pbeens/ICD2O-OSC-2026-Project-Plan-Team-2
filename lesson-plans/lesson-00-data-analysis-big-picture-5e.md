## Learning Goals

Students will be able to:

| - Define data analysis in beginner-friendly language.<br>- Explain why data analysis is used for decisions in real-world contexts.<br>- Describe the IPO model (Input, Process, Output) and apply it to a data workflow.<br>- Classify beginner tasks as Input, Process, or Output.<br>- Connect data evidence to decision-making. |

## Lesson Body

| This lesson builds shared vocabulary and a mental model before students code. It sets up the course sequence: Lesson 00 (big picture), Lesson 01 (Jupyter basics), Lesson 02 (extreme heat analysis). |
| :---- |

### Engage

| Present a short scenario: “Should our city add more cooling centres?” Ask students what evidence would be needed before making the decision. |
| :---- |

### Explore

| Students sort classroom examples into Input/Process/Output (for example: importing libraries, loading a CSV, cleaning missing values, plotting a chart, interpreting results). |
| :---- |

### Explain

| Teacher explicitly defines data analysis and the IPO model. Emphasize that analysis is not just making charts; it includes preparing data and reasoning from evidence. |
| :---- |

### Elaborate

| Students map a full mini-workflow to IPO and explain how each step contributes to better decisions. Extension: compare a weak process (no cleaning) vs a strong process (cleaned data). |
| :---- |

### Evaluate

| Gather evidence using observations, conversations, and student products. Exit ticket: define data analysis, define IPO, and classify three steps correctly as Input/Process/Output. |
| :---- |

## Homework

| Write a short real-world example where a decision should be based on data, and identify one Input, one Process, and one Output for that example. |
| :---- |

## Comments

| Foundational lesson for all subsequent notebook lessons. Keep coding minimal; focus on understanding the purpose and structure of analysis. |
