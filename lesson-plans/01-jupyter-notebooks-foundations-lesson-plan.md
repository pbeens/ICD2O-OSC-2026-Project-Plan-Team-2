## Lesson Body

| This lesson teaches notebook mechanics and workflow so students can focus on analysis in Lesson 02. |
| :---- |

### Engage

| Show two screenshots: plain script vs notebook with text + code + output. Ask which format is easier for learning and why. |
| :---- |

### Explore

| Students create a notebook, add one markdown cell and one code cell, then run cells in order. They practise restarting kernel/runtime and re-running top-to-bottom. |
| :---- |

### Explain

| Explain notebook structure (cells), execution order, why imports are grouped at the top, and how outputs reflect the most recently run state. |
| :---- |

### Elaborate

| Students run a short mini-sequence: import libraries, create small DataFrame, inspect `head()`, print column names, and make a tiny chart. |
| :---- |

### Evaluate

| Evidence from observations, conversations, and student products. Checkpoint: student can correctly create markdown/code cells, run all cells in order, and explain one execution-order mistake and fix. |
| :---- |

## Homework

| In 5-7 sentences, explain how Jupyter notebooks help with reproducibility and communication in data analysis. |
| :---- |

## Comments

| Keep this lesson procedural and confidence-building. Use short cells and immediate feedback to reduce beginner frustration. |
